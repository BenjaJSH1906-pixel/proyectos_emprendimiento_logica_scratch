````markdown
# Práctica de Git

## Descripción

Este repositorio tiene como objetivo realizar una práctica de control de versiones con Git.

## Estructura del repositorio

```text
*
├── README.md
└── docs/
    ├── integrante1.txt
    ├── integrante2.txt
    ├── integrante3.txt
    └── integrante4.txt
```

## Actividad

* Crear el repositorio.
* Agregar el archivo `README.md` con la descripción de la práctica.
* Crear la carpeta `docs`.
* Crear cuatro archivos `.txt`, uno por cada integrante.
* Registrar el avance de cada integrante mediante pseudocódigo.
* Guardar los cambios utilizando:
  * `git add`
  * `git commit`
* Repetir el proceso para los cuatro integrantes, manteniendo un historial de cambios en Git.
````
